Angular. Основные термины

Директива
Это класс помеченный декоратором Directive. С помощью нее можно добавлять новое поведение существующему DOM элементу.
Основные черты: в одном DOM элементе может быть много директив. У директив нет View (template). Внутри директивы нельзя использовать пайпы.
Когда использовать: создание масок, dragAndDrop, button и тд.
Компоненты
Это класс помеченный декоратором Component. Созданы для того, чтобы разбивать приложение на маленькие кусочки.
Основные черты: в одном DOM элементе может быть 1 компонент, в отличие от директив. Есть View (template). Внутри компонента используются пайпы.
Когда использовать: создание представления.
Пайпы
Это класс реализующий interface PipeTransform. Нужны для преобразования данных при их выводе в HTML.
Основные черты: Реализует метод transform(value: any, ...args: any[]) и возвращает преобразованное значение.
Когда использовать: курсы валют, форматирование чисел и дат, преобразование Enum в текст.
Частая ошибка - использовать функцию в шаблоне там где нужен Pipe.
Сервисы
Это классы для выпонения ряда задач.
Поставка данных приложению, может обращаться к бекенду.
Может быть каналом связи между несколькими компоннетами.
Может инкапсулирвоать бизнес логику.
Для того, чтобы сервис мог использовать другой сервис или модуль, к классу сервиса применяется декоратор Injectable, этот подход называется DI.
Модули
Базовая составляющая Angular. Любое приложение содержит хотя бы 1 модуль Root (AppModule).
Любая библиотека Angular это набор мудулей, в случае UiKit - кнопка это модуль, Select модуль и тд.
Модули можно лениво подгружать для ускорения загрузки приложения.
Состав модуля:
declarations - классы представлений, которые принадлежат модулю.
exports - набор классов представлений, которые использоваться в других модулях.
imports - модули от которых зависит данный модуль.
providers - классы сервисы используемые модулем.
bootstrap - корневой компонент приложения Angular.
Формы
Есть два варианта взаиможействия данных с формой:
Template-Driven подход основанный на ngModel директиве
Reactive Forms - реактивные формы.
При разработке UiKit стоит учитывать поддержку обоих подходов.
